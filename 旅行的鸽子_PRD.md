## 一、产品概述

### 1.1 产品背景

现代用户在旅行和日常生活中会持续产生照片、语音、文字等碎片内容，但从“随手记录”到“整理并发布”之间仍存在高门槛：素材分散、记忆流失、平台文案成本高，最终导致大量内容停留在“拍了、说了、记了，但没有整理和分享”的状态。

**旅行的鸽子**的目标，是把“记录”与“生成”做成同一条顺滑链路：用户先用最轻量的方式留下片段，再由 AI 帮助补全细节、整理上下文并生成适配不同场景的内容。

### 1.2 当前产品定位

当前项目是一款**本地优先（local-first）的旅行与生活记录 Web 应用**：

- 后端运行在用户本机，使用 `Express + SQLite`
- 前端为无构建步骤的浏览器 SPA
- 用户通过浏览器访问本机服务，可在电脑或局域网手机上使用
- 数据默认保存在本机数据库与媒体目录，不依赖产品方云端账号体系

### 1.3 目标用户

| 用户群体 | 描述 | 核心痛点 |
|---|---|---|
| 旅行爱好者 | 旅途中频繁拍照、记录瞬时感受 | 素材很多，回头很难串成内容 |
| 生活记录者 | 日常探店、散步、通勤中有轻表达需求 | 想发内容但不想花时间组织语言 |
| 轻度创作者 | 偶尔分享，但不做重度运营 | 需要 AI 帮忙补细节、整理结构、出成稿 |

### 1.4 核心价值主张

- **轻输入**：单击写字、长按录音、上滑拍照，尽量不打断当下体验
- **本地沉淀**：记录、媒体、日记都优先存本机，适合个人长期积累
- **AI整理**：AI 先理解记录，再通过简短对话补细节并生成内容
- **多场景输出**：当前支持小红书、朋友圈、日记三种产出方向

## 二、产品目标与阶段判断

### 2.1 当前阶段目标

当前版本的核心目标不是大规模商业化上线，而是**验证“轻记录 → AI补全 → 平台化生成”这条产品主链路是否成立**。

### 2.2 本阶段验收标准

| 目标 | 验收口径 |
|---|---|
| 记录链路可用 | 用户可稳定完成文字、语音、拍照三类记录 |
| 时间轴可管理 | 用户可查看、编辑、删除、多选记录，支持日/周/月浏览 |
| AI链路可闭环 | 选中记录后可完成对话补充、平台选择、内容生成 |
| 本地沉淀成立 | 日记可自动入库、历史可查看与编辑 |
| 配置可自助 | 用户可自行配置 LLM 与语音转写所需 Key / Endpoint / Model |

### 2.3 暂不作为本阶段目标的内容

- 账号体系与多端同步
- 云端托管与 SaaS 化
- 直接跳转三方 App 发布
- 团队协作与共享旅行
- 生成内容历史中心

## 三、产品形态与信息架构

### 3.1 产品形态

```
浏览器（桌面/手机）
    ↓
本地 Express 服务
    ├─ 静态前端 public/
    ├─ REST API /api/*
    ├─ SQLite 数据库 server/pigeon.db
    └─ 媒体文件目录 server/media/
```

说明：

- 默认 HTTP 端口为 `3000`
- 如存在证书，则同时启动 HTTPS `3443`
- 手机端使用麦克风时，优先通过 HTTPS 访问
- 启动时会打印本机与局域网访问地址

### 3.2 页面结构

```
底部 Tab
├── 首页 home
├── 时间轴 timeline
└── 我的 profile

Overlay 页面
├── 照片预览 photoPreview
├── 照片确认 photoConfirm
├── 鸽子助手 assistant
├── 生成结果 result
├── 日记列表 diaryList
├── 日记详情 diaryDetail
├── 模型设置 modelSettings
├── Prompt 列表 promptList
└── Prompt 编辑 promptEdit
```

### 3.3 核心流程总览

```
用户记录（文字 / 语音 / 拍照）
    ↓
系统附加时间、坐标，并异步补全地址
    ↓
时间轴按日/周/月聚合展示
    ↓
用户多选记录 → 如有多张照片则进入照片确认
    ↓
进入鸽子助手
    ├─ AI先开口提问，帮助回忆细节
    ├─ 用户可文字或语音回复
    ├─ 用户可跳过对话，直接选择平台
    └─ 对话中的新增信息会尝试按记录回写 ai_supplement
    ↓
平台选择：小红书 / 微信朋友圈 / 存为日记
    ↓
AI 流式生成内容
    ↓
日记：自动入库并提示查看
小红书/朋友圈：进入结果页，支持复制、优化、重新生成
```

## 四、功能详细说明

### 4.1 首页：轻记录入口

#### 4.1.1 鸽子按钮交互

首页核心是居中的鸽子按钮，承载三种记录方式：

| 手势 | 功能 | 当前实现 |
|---|---|---|
| 单击 | 文字记录 | 弹出输入层，支持多行文字，支持附带最多 3 张照片 |
| 长按 | 语音记录 | 开始录音，松手结束，上滑取消；先保存再后台转写 |
| 上滑 | 拍照记录 | 优先唤起拍照；Android 支持自定义相机兜底，其余走系统文件/相机选择 |

#### 4.1.2 首页辅助体验

- 首次进入会展示三步手势引导蒙层
- 鸽子按钮下方常驻“上滑拍照 · 长按录音”提示
- 首页副标题会展示“今天已记录 N 条旅程片段”

#### 4.1.3 文字记录

文字记录弹层支持：

- 录入正文
- 附带最多 3 张图片
- 保存时获取当前位置坐标
- 先保存记录，再异步回填地址

#### 4.1.4 语音记录

语音记录流程：

1. 浏览器录音
2. 原始音频先上传保存为媒体文件
3. 创建 `voice` 类型记录，初始内容为“待转写”
4. 后端将音频转为 16kHz 单声道 WAV
5. 通过字节跳动 WebSocket ASR 转写
6. 转写结果回写到记录正文

说明：

- 语音转写依赖单独的 `voice_api_key`
- 手机端麦克风要求 HTTPS

#### 4.1.5 拍照记录

拍照后进入照片预览页，支持：

- 预览照片
- 输入文字说明 `caption`
- 按住语音补充并自动转写到说明框
- 保存照片记录及可选语音附件

补充规则：

- 若用户没有填写说明，且照片也没有附带语音说明，系统会尝试调用 AI 自动生成简短中文画面总结
- 单次从首页可批量选择多张照片；这类照片会直接保存为多条 `photo` 记录

---

### 4.2 记录与数据模型

#### 4.2.1 记录类型

当前记录 `type` 仅支持三类：

| 类型 | 含义 |
|---|---|
| `text` | 文字记录，可附带图片 |
| `voice` | 语音记录，正文为转写结果 |
| `photo` | 照片记录，可带说明文字与语音附件 |

#### 4.2.2 记录关键字段

| 字段 | 说明 |
|---|---|
| `record_id` | 记录唯一 ID |
| `type` | 记录类型 |
| `content` | 文字内容或语音转写结果 |
| `media_filename` | 单媒体文件名，兼容旧结构 |
| `media_filenames` | 多图文件名数组，当前最多 3 张 |
| `caption` | 照片说明 |
| `voice_media_filename` | 照片附带语音文件 |
| `created_at` | 创建时间 |
| `latitude` / `longitude` | 记录时坐标 |
| `address` | 逆地理解析后的文本地址 |
| `group_id` | 自动分组 ID |
| `ai_supplement` | 对话后抽取回写的补充摘要 |
| `is_deleted` | 软删除标记 |

#### 4.2.3 自动分组规则

系统会尝试按“时间 + 地点”把相邻记录归为同组：

- 时间窗口：最近 30 分钟内
- 距离阈值：500 米以内
- 两个条件需同时满足

说明：

- 分组能力已经存在于数据层
- 当前前端时间轴主要按日/周/月展示记录本身，未单独做“分组聚合卡片入口”

---

### 4.3 时间轴

#### 4.3.1 浏览维度

时间轴支持三种查看模式：

| 模式 | 当前表现 |
|---|---|
| 日 | 默认模式；按日期折叠，点击展开后查看当天全部记录 |
| 周 | 按自然周折叠，展开后继续按日展示 |
| 月 | 按自然月折叠，展开后继续按日展示 |

补充说明：

- 前端按用户 IANA 时区分日
- 服务端 `/api/records` 与 `/api/records/dates` 也要求带 `tz`

#### 4.3.2 时间轴展示内容

每条记录卡片当前会展示：

- 类型图标
- 时间
- 地址（如已回填）
- 正文或照片说明
- 照片缩略图网格
- `ai_supplement`（如有）

#### 4.3.3 时间轴操作

当前支持以下操作：

| 操作 | 说明 |
|---|---|
| 展开/收起 | 点击日/周/月折叠卡片 |
| 选择模式 | 右上角“选择”进入多选 |
| 批量生成 | 已选记录进入 AI 流程 |
| 批量删除 | 对已选记录软删除 |
| 单条编辑 | 修改文字、照片说明，或重传附带图片 |
| 单条删除 | 删除前二次确认 |

#### 4.3.4 多选进入 AI

- 多选状态下展示底部操作栏
- 若选中记录里存在多张 `photo`，先进入“选择照片”页做二次勾选
- 确认后进入鸽子助手

### 4.4 鸽子助手

#### 4.4.1 角色定位

鸽子助手不是单次“直接出稿”的黑盒，而是一个多阶段 AI 状态机，承担以下职责：

- 理解用户选择的记录
- 主动发问，帮助补全细节
- 在合适时机引导用户选择生成平台
- 对对话内容做补充信息抽取
- 生成目标内容
- 对生成结果做二次优化

#### 4.4.2 状态阶段

当前代码中的阶段为：

```
understanding → chatting → platform → generating → done
```

#### 4.4.3 对话机制

当前行为包括：

- 进入页面后 AI 先开口
- 默认每轮只问一个问题
- 用户可文本回复，也可在助手页点击麦克风录音转写后发送
- 页面顶部与底部都提供“跳过对话，直接选择生成平台”
- 处于平台选择页时，可返回继续对话

#### 4.4.4 补充信息回写

鸽子助手会将用户在对话中新增的有效信息抽取为 JSON，并尝试按记录写回：

- 写回字段：`ai_supplement`
- 写回时机：发送消息后、生成完成后
- 抽取策略：优先按记录 ID 一一映射
- 若是多记录场景且模型未按 JSON 返回，会提示需要保留 `{{records_index}}`

#### 4.4.5 平台选择

当前支持三个目标平台：

| 平台 | 输出特点 |
|---|---|
| `xiaohongshu` | 标题 + 正文 + 标签的图文笔记 |
| `moments` | 口语化、简短的朋友圈文案 |
| `diary` | 详细叙事内容，并自动保存到本地日记 |

---

### 4.5 内容生成与结果页

#### 4.5.1 生成方式

- 当前内容生成通过后端 `/api/ai/chat-stream` 流式返回
- 支持两类上游协议：`Anthropic Messages API` 与 `OpenAI 兼容 chat/completions`
- 同一套助手模型配置同时用于：
  - 对话
  - 平台内容生成
  - 照片说明自动生成
  - 结果页 AI 优化

#### 4.5.2 结果页行为

对小红书和朋友圈：

- 生成完成后进入结果页
- 正文区域可直接编辑
- 顶部支持复制全文
- 底部支持 AI 优化、撤销优化、重新生成

对日记：

- 生成完成后不进入结果页
- 直接写入本地 `diaries` 表
- 弹出“已存入日记”Toast，并提供“点击查看”

#### 4.5.3 AI 优化

结果页支持输入自然语言优化指令，例如：

- “更活泼一点”
- “压缩到 200 字以内”
- “减少 emoji”

系统会基于当前版本内容重新生成，并维护一个版本栈用于撤销。

---

### 4.6 日记系统

#### 4.6.1 日记来源

当前只有一种自动入库方式：

- 用户在鸽子助手中选择“存为日记”

说明：

- 小红书与朋友圈生成结果当前不会自动保存为日记
- 产品中也没有“把现有生成稿一键转存为日记”的独立入口

#### 4.6.2 日记数据结构

| 字段 | 说明 |
|---|---|
| `diary_id` | 日记唯一 ID |
| `title` | 当前默认用日期生成标题 |
| `content` | 日记正文 |
| `record_ids` | 关联的记录 ID 数组 |
| `platform` | 当前固定为 `diary` |
| `created_at` / `updated_at` | 创建与更新时间 |

#### 4.6.3 图片嵌入方式

日记正文支持用 `[[image:filename]]` 标记图片位置：

- 生成日记时会自动把选中记录中的图片插入正文
- 日记列表预览时会忽略图片标记，仅显示纯文本摘要
- 日记详情页支持查看与编辑

#### 4.6.4 日记当前能力边界

已实现：

- 日记列表
- 日记详情
- 日记编辑
- 日记删除

未实现：

- 导出 TXT / Markdown
- 标签、分类、搜索

| 分组 | 当前内容 |

#### 4.7.3 未完全落地项

### 4.8 模型设置与 Prompt 设置

#### 4.8.1 模型设置

当前设置页分为两套配置：

| 配置区 | 当前字段 |
|---|---|
| 语音转文字 | `voice_api_key` |
| AI 助手 | `assistant_provider`、`assistant_endpoint`、`assistant_api_key`、`assistant_model`、`assistant_temperature`、`assistant_max_tokens` |

关键说明：

- 语音转写并不是 OpenAI Whisper 直传模式，而是**后端代理到字节跳动大模型 ASR**
- AI 助手配置同时用于对话、生成、优化和照片说明

#### 4.8.2 Prompt 设置

系统当前维护 6 个 Prompt：

| 编号 | 名称 | 用途 |
|---|---|---|
| P1 | 鸽子助手对话 Prompt | 指导 AI 如何发问与引导 |
| P2 | 补充摘要提炼 Prompt | 对话信息抽取并按记录回写 |
| P3 | 小红书生成 Prompt | 生成小红书结构化文案 |
| P4 | 朋友圈生成 Prompt | 生成朋友圈短文案 |
| P5 | 日记生成 Prompt | 生成日记正文 |
| P6 | 内容优化 Prompt | 对结果页内容做定向修改 |

当前设置能力：

- 查看系统默认 Prompt
- 自定义覆盖
- 重置为默认

当前未实现：

- 在线测试入口

## 五、后端与接口约束

### 5.1 核心后端模块

| 模块 | 说明 |
|---|---|
| `server/index.js` | Express 入口，同时启动 HTTP/HTTPS |
| `server/db.js` | SQLite 数据层，包含 Records / Groups / Diaries / Settings / Prompts |
| `server/routes/records.js` | 记录查询、创建、更新、删除、逆地理解析 |
| `server/routes/diaries.js` | 日记 CRUD |
| `server/routes/settings.js` | 设置读写 |
| `server/routes/prompts.js` | Prompt 管理 |
| `server/routes/media.js` | 媒体上传与访问 |
| `server/routes/ai.js` | AI 聊天流、非流式聊天、语音转写 |

### 5.2 核心接口

| 接口 | 用途 |
|---|---|
| `GET /api/records` | 按日期 + 时区查询记录 |
| `GET /api/records/dates` | 查询所有有记录的日期 |
| `GET /api/records/all-active` | 时间轴全量拉取 |
| `POST /api/records` | 创建记录 |
| `PUT /api/records/:id` | 更新记录 |
| `PUT /api/records/:id/supplement` | 回写 AI 补充信息 |
| `DELETE /api/records/:id` | 软删除记录 |
| `POST /api/media/upload` | 上传图片/音频 |
| `GET /api/media/:filename` | 读取媒体文件 |
| `POST /api/ai/chat-stream` | 流式聊天/生成 |
| `POST /api/ai/chat` | 非流式 AI 请求 |
| `POST /api/ai/transcribe` | 音频转写 |

### 5.3 时区与地址策略

- 日期查询必须以用户 IANA 时区为准
- 逆地理解析优先用在线服务回填地址
- 地址属于异步增强信息，不阻塞记录创建

## 六、非功能性要求与限制

### 6.1 隐私与数据

- 记录数据、日记、设置默认保存在本机 SQLite
- 图片与音频保存在本地媒体目录
- 不依赖产品账号体系
- 用户填写的第三方 API Key 保存在本地设置表

注意：

- 当前并未实现“本地加密存储 API Key”
- 因此 PRD 不应把“已加密”写成既成事实

### 6.2 网络依赖

当前版本不是完全离线应用：

| 功能 | 是否依赖网络 |
|---|---|
| 文字记录 | 否，本地服务可用即可 |
| 拍照/保存媒体 | 否，本地服务可用即可 |
| 浏览时间轴/日记 | 否，本地服务可用即可 |
| 地址解析 | 是 |
| 语音转写 | 是 |
| AI 对话与生成 | 是 |

### 6.3 浏览器与设备限制

- 手机麦克风在局域网下需 HTTPS
- Android 额外提供了自定义相机兜底方案
- 中文输入法需要兼容 `compositionend` 与 `keyup`
- 页面高度需兼容移动端 `100dvh`

## 九、关键产品决策

## 十、术语表

| 术语 | 定义 |
|---|---|
| 记录（Record） | 用户一次输入形成的最小内容单元 |
| 时间轴（Timeline） | 按日/周/月查看全部记录的主页面 |
| 鸽子助手（Assistant） | 负责提问、补充、平台选择、生成与优化的 AI 模块 |

| 结果页（Result Page） | 小红书/朋友圈生成后的编辑与优化页面 |
| 日记（Diary） | 本地存储的长文本沉淀，可含图片标记 |
| Prompt | 控制 AI 行为的模板文本，当前共 6 个 |
| Local-first | 以后端、本地数据库、本地媒体目录为主要存储载体的产品形态 |
